# Contexto del proyecto — MetalShop

Ecommerce en Python (FastAPI) con frontend estático y estética metálica premium.

Lee y aplica los estándares base del proyecto: @docs/base-standards.md

## Flujo de trabajo

Spec-Driven Development (OpenSpec). Todo cambio recorre proposal → specs → design → tasks →
apply → archive antes y durante la codificación. Ver `README.md` y `openspec/config.yaml`.

## Estándares por área

- `docs/base-standards.md` — principios base, idioma, skills, planificación, reglas OpenSpec.
- `docs/documentation-standards.md` — estructura y mantenimiento de la documentación.
- `docs/backend-standards.md` — Python / FastAPI / SQLAlchemy / pytest / Ruff.
- `docs/frontend-standards.md` — HTML/CSS/JS vanilla y paleta metálica.

## Contexto del producto

- `SPEC.md`, `PLAN.md`, `TASKS.md` — línea base funcional v1 (ya implementada).
- `openspec/` — cambios vigentes y specs (fuente de verdad para mejoras y construcción faltante).
